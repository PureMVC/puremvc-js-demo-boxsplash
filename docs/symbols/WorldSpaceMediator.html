Class WorldSpaceMediator


Extends Mediator.
The Mediator subclass attached to the WorldSpace View. Its primary responsibilities here are to handle notification from the system that require changing the state of the animation (BoxConfigVO data) This Mediator is registered by the ShellMediator's constructor.
Defined in: boxsplash.js.

Handles notifications broadcasted by the system provided the notification is listed in the listNotificationInterests() return value.
Parameters:
{Notification} notification
The notification to act upon.
{String[]} listNotificationInterests()
Provides a list of notification interests to the View. Without an accurate list, the handleNotification() method will may be invoked. A common mistake by developers is to provide handling routines in the handleNotification() method but forget to add the notification name in the listNotificationInterests() array.

Note that changing this array at runtime will not have any affect on notification interests since this method is called by the View a single time when the Mediator is first registered.

Returns:
{String[]} the array of notification names to act upon.
